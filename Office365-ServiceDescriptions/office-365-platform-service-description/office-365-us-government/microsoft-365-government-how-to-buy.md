米国の公共部門の独自の要件と進化する要件に対応して、マイクロソフトは米国の公共部門のお客様向けに特定のMicrosoft 365政府環境を作成しました。 このサービスの説明では、環境の概要と購入方法の詳細を説明します。 このサービスの説明は、他の[Microsoft 365やOffice 365サービスの説明と](../../office-365-service-descriptions-technet-library.md)一緒に読むことをお勧めします。
  
## <a name="whats-microsoft-365-government"></a>政府Microsoft 365何ですか?

Microsoft 365政府は、管理された未分類の情報を保持するために後援する米国政府機関や請負業者に合わせた生産性、セキュリティ、モビリティ クラウド ソフトウェア機能のセットです。 最も厳しいコンプライアンス要件を満たす独自の環境を通じて提供されるMicrosoft 365政府は、マイクロソフトの商用クラウド エンタープライズ製品の機能と機能にできるだけ密接に一致する米国政府のお客様向けのクラウド サービスです。

## <a name="what-makes-microsoft-365-government-different-from-microsoft-365-for-enterprise-offerings"></a>Microsoft 365政府が企業向けMicrosoft 365と違う理由は何ですか?

Microsoft 365政府は、次の4つの主要分野における米国政府機関のコンプライアンス要件に対応するように独自に設計されています。

**排他的なコミュニティ。** 政府機関のデータ センターは、政府機関または部門、または米国政府に代わって管理された情報を保持および処理する権限を与えられた商業会社のみが利用できます。

**審査を受けた人員。** 組織の顧客コンテンツへのアクセスは、米国市民であり、必要な身元調査に合格した Microsoft 担当者のスクリーニングに制限されます。
サードパーティの監査。 すべてのインフラストラクチャは、連邦政府機関が運営許可(ATO)を発行するために使用するセキュリティ評価報告書または認証書を提供する認定第三者監査人によって監査されます。 

**コンテンツストレージ。** データは、準拠したインフラストラクチャを持つ米国大陸内に格納されます。 次の表は、政府の環境とコミットメントのMicrosoft 365をまとめたものです。<br><br>

|:----------|:----------------------------------|
|Microsoft 365政府GCC  <br/> |フェドランプハイ、DFARS、CJIS、IRS 1075、ディサSRG L2 <br/> |
|Microsoft 365政府GCC高  <br/> |フェドランプハイ、イタル、DFARS、DISA SRG L4コントロール <br/> |
|Microsoft 365政府 DoD  <br/> |ディサ SRG L5  <br/> |

マイクロソフトのセキュリティおよびコンプライアンス認定の詳細については、マイクロソフト セキュリティ[センター](https://www.microsoft.com/trustcenter/default.aspx)を参照してください。  

## <a name="microsoft-365-government-eligibility-and-validation"></a>Microsoft 365政府の適格性と検証

Microsoft 365政府は、管理された情報を保持または処理するために後援する適格な政府の顧客および非政府組織に利用可能です。 これら 2 つのカテゴリのお客様の適格性基準は、Microsoft 政府機関クラウド全体で一貫しています。

GCCとGCC高の両方の製品は、マイクロソフト政府機関クラウドの対象となるすべてのお客様に利用可能であり、DoD 環境は米国国防総省専用です。 サービスの可用性と価格が異なり、GCCフェドランプ高またはDoD制御未分類情報(CUI)を保持していないすべての顧客のためのヒーローの提供のままです。

適格な政府の顧客は、次の 1 つによって定義されます。

- 連邦政府機関で、米国政府の局、オフィス、代理店、部門、またはその他の団体として定義されています。
- 次のいずれかとして定義される状態/ローカル エンティティ。
    - 米国の州または地方自治体の機関。
    - 米国の郡、自治区、連邦、都市、自治体、町、町、特別目的地区、または顧客の州の法律によって確立され、顧客の州の管轄区域および地理的境界内に位置するその他の同様の種類の政府の器用性
    - コロンビア特別区、プエルトリコ連邦、グアム、アメリカ領サモア、アメリカ領バージン諸島、北マリアナ諸島
- インドの部族としての地位、またはアラスカ州のネイティブビレッジまたはアラスカ地域ネイティブ企業のおかげで、米国内務省からの資金提供とサービスの対象となる連邦政府が認めた部族のエンティティとして定義される部族のエンティティ

次の種類のデータを保持し、記載されている証明を提供できる非政府機関では、Microsoft 政府機関クラウド検証プロセスの対象となります。

規制の対象となるデータを含む商業用の個人団体。 受け入れ可能な政府のデータ型は次のとおりです。
- 武器の国際トラフィック(ITAR)
- 制御未分類情報 (CUI)
- 国防総省(DoD)未分類の制御核情報(UCNI)
- エネルギー省 (DoE) UCNI
- 刑事司法情報(CJI)
- 国防総省の影響レベルデータ
- 政府Microsoft 365必要なその他の種類のデータ

米国の子会社を通じてサービスを購入するには規制されたデータが必要な場合がありますが、国際的な商業団体が適格である場合があります。

上記のグループのメンバーシップの証明は、Microsoft 365政府の提供へのアクセスのために必要とされます。
